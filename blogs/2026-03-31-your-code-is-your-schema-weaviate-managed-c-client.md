Rename a property, forget to update your config, and you've got a runtime error.</p><p>Enter the <a href="https://github.com/weaviate/csharp-client-managed" rel="noopener noreferrer" target="_blank"><strong>Weaviate Managed .NET Client</strong></a>. As a newly released alternative to traditional client libraries, it brings an Entity Framework Core-inspired experience directly to Weaviate. You can now define collections with C# attributes, query with type-safe <a href="https://learn.microsoft.com/en-us/dotnet/csharp/linq/" rel="noopener noreferrer" target="_blank">LINQ</a> expressions, and let the client handle schema creation, migrations, and object mapping—all in idiomatic .NET.</p><p>To show what this looks like in practice, we'll build a semantic document search engine. Along the way, you'll see how attributes replace separate schema config objects, how LINQ-style expressions compile into <a href="https://weaviate.io/blog/what-is-a-vector-database" rel="noopener noreferrer" target="_blank">vector search</a> filters, and how a single codebase can grow from personal tool to multi-tenant platform.</p><div class="theme-admonition theme-admonition-caution alert alert--warning admonition_LlT9"><div class="admonitionHeading_tbUL"><span class="admonitionIcon_kALy"><svg viewBox="0 0 16 16" xmlns="http://www.w3.org/2000/svg"><path d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z" fill-rule="evenodd"></path></svg></span>Technical Preview</div><div class="admonitionContent_S0QG"><p>The Weaviate Managed .NET Client is out as a technical preview.
